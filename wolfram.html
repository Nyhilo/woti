Wolfram Code and the Rules of Cellular Automation

How to Use This App

Set the rules by clicking on the outlined boxes under the Rules section.

Then, you can click on any of the cells in the Input Row to switch their state.

Finally, the buttons above the Input Row all do different things. Experiment with different values and see what happens!

Introduction

Elementary Cellular Automation is a simple model of computation that lets us visualize the interactions between "cells" (or pixels, or squares) in a grid. The way it works is like this:

  1. For each cell in the grid, look at the cells directly above, above and to the left, and above and to the right.
  2. Depending on the rule defined for those 3 cells, determine whether the cell we are looking at is "alive", meaning colored in, or "dead", meaning left blank.

The Rules

The number in parentheses is the Wolfram Code for the current ruleset. Generally, the rules for a cellular automation system is shown in a table like so:

111 110 101 100 011 010 001 000
0 0 0 1 1 1 1 0

The 1s in this case represent cells that are alive and the 0s represent that they are dead. Using a table this, you can determine the "rule" for whether the cell below should be alive or dead.

However, that format can be a little hard to visualize, so this tool uses a cluster of 4 boxes to represent the rules in a visual way.

This means, "If the cell to the top left of the current cell is dead, and the cell above and to the top right are alive, make the current cell alive.

The app here then follows those rules for every cell in the simulation area, showing us the pattern that the set of rules creates.
